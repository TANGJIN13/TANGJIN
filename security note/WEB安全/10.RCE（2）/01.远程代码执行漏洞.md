# 一、代码执行的含义
代码执行是指服务器上的解释器按照程序编写的指令和算法逐步执行代码的过程。

# 二、代码执行漏洞
由于应用程序在调用一些能够将字符串转换为代码的函数（如PHP中的eval）时，没有考虑用户是否控制这个字符串，则会导致代码执行漏洞的发生。

# 三、漏洞危害
1.控制受害者的系统：攻击者可以利用漏洞控制受害者的系统，并对其进行各种恶意操作，如上传、下载、删除、修改文件等，甚至可以利用该漏洞构建僵尸网络、挖掘加密货币等。
2.窃取敏感信息：攻击者可以通过该漏洞窃取受害者系统中的敏感信息，如账号、密码、财务信息等。
3.进行后门植入：攻击者可以利用漏洞在系统中植入后门，以便在未来任何时间进行下一步的攻击。

# 四、PHP代码执行函数
## 1.eval
- 格式：`eval(string $code)`
- 作用：字符串按照PHP代码来计算。该字符串必须是合法的PHP代码，且必须以分号结尾。
 - 示例：
```
 <?php @eval($_POST['a']);?>
```

## 2.assert
- 格式：`assert ( mixed $assertion [, string $description ] )`
- 作用：如果assertion是字符串，它将会被assert()当做PHP代码来执行。
- 示例：
```
 <?php @assert($_POST['a'])?>
```

注意：assert()函数是直接将传入的参数当成PHP代码执行，可以不需要以分号结尾，传入的代码也可以不加分号结尾

## 3.preg_replace
- 格式：`preg_replace ( mixed $pattern , mixed $replacement , mixed $subject [, int $limit = -1 [, int &$count ]] )`
- 作用：执行一个正则表达式的搜索和替换，搜索subject中匹配pattern的部分，以replacement进行替换。
- 示例：
```
<?php
echo(preg_replace("/test/",$_POST["a"],"just test"));
?>
```

当第一个参数使用/e作为修饰符，会把第二个参数传入的字符串当作php代码执行。
```
<?php preg_replace("/abc/e", $_REQUEST['cmd'], "abc"); ?>
```
php>5.5不支持

## 4.all_user_func
- 格式：`call_user_func ( callable $callback [, mixed $parameter [, mixed $... ]] )`
- 作用：把第一个参数作为回调函数调用，第一个参数callback是被调用的回调函数，其余参数是回调函数的参数。
- 示例：
```
<?php
call_user_func("assert",$_POST['cmd']);
?>
```

# 五、漏洞防御措施
1.尽量不要使用危险函数
2.对数据进行黑白名单处理
3.对传入的特殊字符转义
4.谨慎使用preg-replace的e修饰符
