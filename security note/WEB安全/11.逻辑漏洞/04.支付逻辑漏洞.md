# 一、漏洞简介
支付逻辑漏洞是指在支付系统中存在的安全漏洞或错误的逻辑流程，使攻击者能够以非法或欺诈性的方式获得支付的利益或优势。这些漏洞可能存在于支付网关、电子商务平台、移动支付应用程序或其他与支付过程相关的系统中。

# 二、漏洞危害
支付漏洞通常都是篡改价格。比如，一分钱买任何东西。少收款、企业收费产品被免费使用，直接造成企业的经济损失。

# 三、测试方法
修改购买数量：在进行支付订单的时候，可以修改物品的数量来进行操作，可以通过支付一件的价格购买多件，或者修改成负数进行增加资金、修改支付价格、修改支付对应的商品、修改支付的状态、修改附属优惠、测试数据包未删除、修改支付接口、重复支付、最大额支付等。

# 五、漏洞修复
1.敏感参数使用密文传输；
2.服务端校验过滤用户提交的参数；
3.提交订单时后台判断单价是否与数据库中相符；
4.服务端计算金额是判断是否为正数；
5.生成数据签名，对用户金额和订单签名。
